invoke
(/datum/callback/on_completion)

Call wrapper for executing the task Very important note that the whole thing relies on this single one NOT crashing as it's our mechanism for handling sleeps, or crashes within task implements. This ensures something is always returned eventually and tasks are not left dangling.

Intended usage is to call it from a waitfor=0 wrapper, passing a callback If the task sleeps, invoke() early-returns NM_TASK_ASYNC and keeps following it Once it effectively finishes, the provided callback receives result.

Do note that on_completion is completion of invocation, not task
